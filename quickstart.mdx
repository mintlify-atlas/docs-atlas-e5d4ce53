GBrain doesn't ship with demo data — it finds your markdown and makes it searchable. This guide walks you from a fresh install to running your first hybrid search query against your own files.

<Note>
**No Postgres yet?** The GBrain knowledge model — the skills, schema, and compiled truth + timeline pattern — works with any AI agent that reads and writes markdown files. You don't need Postgres to start. Add it when `grep` stops being fast enough.
</Note>

## Prerequisites

You need three things for full functionality:

| Dependency | Purpose | Required? |
|---|---|---|
| **Supabase Pro** ($25/mo) | Postgres + pgvector database | Yes — for search |
| **OpenAI API key** | Embeddings (`text-embedding-3-large`) | No — keyword search still works without it |
| **Anthropic API key** | Multi-query expansion + LLM chunking (Haiku) | No — search still works without it |

```
   restructure my knowledge base

   with the production agent patterns

7. Set up a daily cron for `gbrain check-update`.
```

## Manual setup

<Steps>
  <Step title="Install GBrain">
    Install globally via bun:

    ```bash
    bun add -g github:garrytan/gbrain
    ```

    Verify the install:

    ```bash
    gbrain --help
    ```
  </Step>

  <Step title="Set API keys">
    ```bash
    export OPENAI_API_KEY=sk-...
    export ANTHROPIC_API_KEY=sk-ant-...
    ```

    Add these to your shell profile (`.zshrc`, `.bashrc`) so they persist across sessions.
  </Step>

  <Step title="Connect your database">
    Run the setup wizard:

    ```bash
    gbrain init --supabase
    ```

    The wizard checks for the Supabase CLI, offers auto-provisioning, and falls back to a manual connection URL if needed. It then runs the full schema migration (tables, indexes, triggers, pgvector extension) and verifies the connection.

    <Warning>
    Use the **Shared Pooler** connection string — not the project URL, not the anon key, not the REST endpoint. In your Supabase project: click **Get Connected** → **Direct Connection String** → **Session Pooler** → copy the **Shared Pooler** string.
    </Warning>
  </Step>

  <Step title="Discover and import your markdown">
    GBrain scans for markdown repos automatically:

    ```
    === GBrain Environment Discovery ===

      ~/git/brain (2.3GB, 342 .md files, 87 binary files)
        Type: Plain markdown (ready for import)

      ~/Documents/obsidian-vault (180MB, 1,203 .md files, 0 binary files)
        Type: Obsidian vault (wikilink conversion available)

    === Discovery Complete ===
    ```

    Import your chosen directory:

    ```bash
    gbrain import ~/git/brain/
    # Imported 342 files into Supabase (1,847 chunks). Embedding in background...
    ```

    To import fast and embed later:

    ```bash
    gbrain import ~/git/brain/ --no-embed
    gbrain embed --stale  # run once embeddings are needed
    ```
  </Step>

  <Step title="Check stats">
    ```bash
    gbrain stats
    # Pages: 342, Chunks: 1,847, Embedded: 1,847, Links: 0
    ```

    Embedded count climbs as background embedding completes (~10–15 min for 7,000 pages).
  </Step>

  <Step title="Run your first search">
    ```bash
    gbrain query "what do we know about competitive dynamics?"
    # 3 results, scored by hybrid search (vector + keyword + RRF fusion)

    gbrain query "what are our biggest risks right now?"
    # Finds pages about moats, board prep, strategy -- by meaning, not keywords
    ```

    Use `gbrain search` for fast keyword-only search, `gbrain query` for full hybrid search with vector + expansion.
  </Step>
</Steps>
